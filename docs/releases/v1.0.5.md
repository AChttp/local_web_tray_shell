# Switch v1.0.5

Maintenance release focused on audit-driven correctness fixes and sidebar/usability improvements.

## Highlights

- Per-command working directory and environment variables, so local services can run from their own repo folder with their own `PORT`, `NODE_ENV`, etc.
- Draggable splitter between the command and site sections of the sidebar; the layout ratio is saved.
- Single-instance enforcement: a second launch exits silently instead of hitting the in-use WebView2 user-data folder and showing a confusing error.
- "Stop all" now asks for confirmation (sidebar button and tray menu) before hard-killing running commands.
- Site cards show a green/red health dot from a periodic probe, so a dead service is visible at a glance.
- Sidebar keyboard navigation: Up/Down moves selection, Enter activates, Delete removes (with the existing confirm).
- Restart action exposed as a 5th command action button (uses the existing restart engine).
- Import/export the full configuration to/from a JSON file.
- Data-safety hardening: corrupt configs are backed up instead of being silently overwritten with defaults, config writes are atomic, and site-URL dedup is consistent between the dialog and the persistence layer.
- Resource and process-lifecycle fixes: WebView2 controls are torn down on exit and on site deletion; `Process` objects are disposed at every transition; the exit path kills process trees synchronously so no child is orphaned.
